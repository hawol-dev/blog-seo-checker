# PostKit — 네이버 블로그 SEO 분석기 디자인 스펙

> **작성일**: 2026-03-24
> **프로젝트**: PostKit
> **범위**: Phase 1 MVP 웹 UI 디자인

---

## 1. 제품 개요

네이버 블로그 글의 SEO 점수를 분석하고 개선 가이드를 제공하는 웹 도구.

- **타겟 유저**: 네이버 블로그 운영자 (상위노출 목표)
- **핵심 플로우**: 글 제목 + 키워드 + 본문 입력 → SEO 점수 (100점) + 개선 가이드
- **수익 모델**: 무료 3회/일, Pro $7.99/월 (무제한 + 히스토리)
- **Phase 1 전략**: 네이버 전용으로 포지셔닝, 추후 구글/티스토리 확장 가능

## 3. SEO 점수 체계 (D.I.A. 기반, 100점)

네이버 D.I.A. (Deep Intent Analysis) 알고리즘 기반 5개 카테고리, 13개 항목.

### 3.1 키워드 최적화 (25점)

| 항목 | 배점 | 기준 |
|------|------|------|
| 제목 내 키워드 | 10 | 제목에 타겟 키워드 포함 + 앞부분 배치 |
| 본문 키워드 밀도 | 8 | 1~3% 적정, 과다 시 어뷰징 감점 |
| 첫 문단 키워드 | 7 | 상위 100자 이내 키워드 등장 여부 |

### 3.2 콘텐츠 구조 (25점)

| 항목 | 배점 | 기준 |
|------|------|------|
| 글자수 | 10 | 1,500자 미만 감점, 2,000~5,000자 최적 |
| 소제목 사용 | 8 | 2개 이상 소제목 구조화 |
| 문단 길이 | 7 | 2~4줄 짧은 문단 유지 (모바일 가독성) |

### 3.3 미디어 활용 (20점)

| 항목 | 배점 | 기준 |
|------|------|------|
| 이미지 수 | 8 | 최소 3장, 글자수 대비 적정 비율 |
| 이미지 간격 | 7 | 300~500자마다 1장 균등 배치 |
| 동영상 포함 | 5 | 1개 이상 포함 시 가산 |

### 3.4 링크 & 태그 (15점)

| 항목 | 배점 | 기준 |
|------|------|------|
| 외부 링크 | 7 | 0개가 만점, 있으면 감점 (네이버 특수) |
| 내부 링크 | 4 | 자기 블로그 내 다른 글 1~2개 |
| 태그 | 4 | 5~10개, 키워드 연관성 |

### 3.5 마무리 & 가독성 (15점)

| 항목 | 배점 | 기준 |
|------|------|------|
| 맺음말 | 5 | 글 마무리 문단 존재 |
| 제목 길이 | 5 | 15~30자 적정 |
| 전체 가독성 | 5 | 줄바꿈 빈도, 이모지/특수문자 과다 여부 |

### 3.6 등급 체계

| 등급 | 점수 | 의미 |
|------|------|------|
| S | 90~100 | 상위노출 준비 완료 |
| A | 75~89 | 거의 다 왔어요 |
| B | 60~74 | 조금만 더 개선하면 |
| C | 40~59 | 개선이 필요해요 |
| D | 0~39 | 많이 부족해요 |

## 4. 화면 설계

### 4.1 공통 디자인 시스템

- **테마**: 다크 모드 (`#08080c` 베이스)
- **폰트**: Plus Jakarta Sans (숫자/영문) + Noto Sans KR (한글)
- **Accent**: `#9b7cfa` (보라)
- **상태 컬러**: Green `#34d399` / Yellow `#fbbf24` / Red `#f87171`
- **레이아웃**: `max-width: 1400px`, 중앙 정렬
- **패널**: `border-radius: 12px`, 패널 간 `gap: 12px`, main `padding: 12px 16px`
- **스크롤 정책**: 페이지 스크롤 없음 (100vh 고정), 내부 컨텐츠만 스크롤

### 4.2 헤더 (모든 페이지 공통)

```
[PostKit 로고 + "네이버 블로그 전용" 뱃지] — [새 분석 | 히스토리] 탭 (정중앙) — [2/3 무료 뱃지 + 아바타]
```

- 높이: 56px 고정
- 하단 accent glow 라인
- 탭은 `position: absolute; left: 50%; transform: translateX(-50%)`로 정중앙

### 4.3 분석 페이지 (핵심)

**레이아웃**: 좌우 분할 (좌 44% 입력 / 우 56% 결과)

**왼쪽 패널 (입력)**:
1. 글 제목 (input)
2. 타겟 키워드 (input, focus 시 accent glow)
3. 본문 내용 (textarea, `flex: 1`, **유일한 스크롤 영역**)
4. 메타 카드 5열 (이미지/동영상/태그/외부링크/내부링크)
5. "네이버 SEO 분석하기" CTA 버튼 (accent gradient, hover 시 glow + lift)

**오른쪽 패널 (결과)**:
1. 점수 헤더: 원형 프로그레스 링 (conic-gradient) + 메시지 + 부가정보
2. 체크리스트 (단일 리스트, 우선순위 순):
   - 수정 필요 (빨강 배경, ✕ 아이콘, 힌트 + "+N점" 뱃지)
   - 개선 추천 (노랑 배경, △ 아이콘, 힌트 + "+N점" 뱃지)
   - 통과 (기본 배경, ✓ 아이콘, 이름 + 점수만)
3. 체크리스트만 `overflow-y: auto`

**분석 범위**: 텍스트 + 사용자 입력 메타데이터 (이미지 수, 동영상 수, 태그 수, 링크 수)

### 4.4 히스토리 페이지

**레이아웃**: 좌우 분할 (좌 50% 리스트 / 우 50% 상세)

**왼쪽 패널 (리스트)**:
- 검색 박스 + 등급 필터 (전체 / S·A / B↓)
- 분석 리스트 (점수 원형 + 제목 + 날짜 + 키워드 + 등급 뱃지)
- 선택된 아이템 하이라이트 (accent border)
- 리스트 내부 스크롤

**오른쪽 패널 (상세)**:
- 선택된 글 제목/메타
- 점수 링 + 메시지
- 이슈 목록 (수정/개선/통과 — 분석 페이지와 동일 포맷)
- "이 글 다시 분석하기 →" 버튼

### 4.5 로그인 페이지

- 전체 화면 중앙에 카드 (420px)
- 배경: accent + green 글로우
- 구성:
  1. PostKit 로고 + "네이버 블로그 전용 SEO 분석기" 뱃지
  2. 피쳐 태그 (13개 항목 / D.I.A. 기반 / 무료 3회)
  3. "로그인하고 분석 시작하기" + 부가 설명
  4. 네이버 로그인 버튼 (초록, 상단) — 타겟 유저가 네이버 블로거이므로 우선 배치
  5. 카카오 로그인 버튼 (노랑)
  6. "또는" 구분선
  7. "로그인 없이 바로 분석하기 (저장 불가)" — 비회원 진입점
  8. 이용약관/개인정보처리방침 링크

## 5. 반응형

- **768px 이하**: 좌우 분할 → 세로 스택, 각 섹션 독립 스크롤
- **모바일**: 어제 브레인스토밍에서 확정된 3화면 스텝 플로우 (입력 → 점수 → 가이드)

## 6. 핵심 Server Actions & Supabase 연동

| 함수 | 위치 | 설명 |
|------|------|------|
| analyzePost() | Server Action | 글 텍스트 + 키워드 + 메타 → SEO 점수 |
| getHistory() | Server Action | Supabase에서 내 분석 히스토리 조회 |
| getAnalysis(id) | Server Action | 특정 분석 결과 상세 조회 |
| getTodayUsage() | Server Action | 오늘 분석 횟수 (무료 3회 제한) |
| Supabase Auth | @supabase/ssr | 네이버/카카오 OAuth (SSR 쿠키 관리) |
